Patient Ledger - Payments 

The Payments card in the Patient Record displays all patient-related payments. Here, users can manage all patient payments which include adding, editing, voiding, refunding, and allocating payments to charges. 

Payments

Add A Payment

1.  Select Add New Payment (in blue).

2.  From the Add Payment slideout, select a Patient or Payer Payment.

  • Patient - payment received from the patient, or for the patient on behalf of the patient's guarantor 
  • Payer - payment received from the patient's designated insurance payer

3.  From the Add Patient Payment slideout, fill in all required fields including:

  • Payment Type - select a payment code related to patient or payer payments
  • Amount - payment amount for the payment type selected
  • Payment Method
    • Cash - patient payments only 
    • Check
    • Direct Deposit - payer payments only 
    • Debit or Credit Card 
  • Transaction Date - payment date

4.  Select POST (in blue).

5.  From the Posted Payment slideout, you have the ability to:

  • Void Payment - allows you to void the payment
  • Allocate - allows you to allocate this payment to an open charge
  • Print - opens up a printable receipt in a new web browser tab
  • Close - returns the user to the patient's payments

6.  When there are multiple charges that are more than the amount of the payment, you will need to allocate the payment to specific charges.

7. Select the checkmarks for the charges you want to allocate.  Then scroll to the right and add the amount of the payment. Select ALLOCATE when you are done.

6.  The payment will now display within the patient's chart, updating both the Payments and Balance card accordingly. 

Process Payment Electronically

Payments processed electronically require Merchant Account and Card Reader set up first. Click here to learn more about Merchant Accounts. 

1.  From the Add Payment slideout, select Payment Method of Debit or Credit Card.

  • Card Swipe - allows you to process the payment via card swipe 
  • Manual Card Entry - allows you to manually enter the card information 
  • Card On File - allows you to use the card save on file

Manual Card Entry

1b.  From the Card Details, select the Merchant Account, enter the card details, followed by SUBMIT (in blue).

Card on File

A Card first needs to be added in the Patient Record Billing section. Click here to learn more about Patient Record - Billing.

1c.  From Merchant Account and Card Details, select the Merchant Account and Card On File, followed by SUBMIT (in blue).

Process Payment Manually

1.  From the Add Payment slideout, select Payment Method of Debit or Credit Card followed by Process Manually (in blue). 

2.  Enter the Last 4 Digits of Card, followed by POST (in blue). 

Payment Details

The Payment details page displays all payment-related information along with actions that can be taken against that payment.

1. Select the Payment you wish to view.

2.  From the Payment details page, view the available actions indicated (in blue):

  • Void - voids the payment
  • Refund Payment - refunds the payment
  • Allocate Payment - allocates the payment to related open charges
  • Receipt - opens the payment receipt in a new tab to print
  • Edit Icon - allows you to edit specific payment details

Note: Actions highlighted in blue indicate that they are available to be taken against the payment in context. Any actions that are not available for the payment in context will be greyed out. This is contingent on payment type (patient or payer) and payment status (unallocated, allocated, voided, refunded).

Edit Payment

1. Select the payment you wish to edit.

2.  From the Payment details screen, select the edit icon (in blue).

3.  From the Update Payment slideout, edit the necessary fields. Click SAVE (in blue).

Note: Editable fields are dependent on the payment method (cash, check, direct deposit, debit or credit card, or ERA) and whether or not all or part of the payment has been allocated.

  • Payment Type -always editable
  • Payer - editable for payer payments only; only editable if none of the payment has been allocated
  • Amount - editable if none of the payment has been allocated
  • Payment Method - editable unless the payment was made with a debit or credit card and if none of the payment has been allocated
  • Transaction Date - editable unless the payment was made with a debit or credit card
  • DOS From - always editable; only available if it is a payer payment
  • DOS To - always editable; only available if it is a payer payment
  • Reference ID - always editable; only available if it is a payer payment
  • AUTH/CHECK/EFT - editable if the payment was made by check, direct deposit, or ERA
  • Card Sale ID - never editable; only viewable if the payment was made with a debit or credit card

Note: In order to update the payment method to a card transaction, select Debit or Credit Card in Payment Method, then proceed through the steps of making the payment.

Void Payment

Voiding a payment from the Payments card can be done in either of two ways.

1a. Select the checkbox of the payment you wish to void, then click the action menu (ellipsis).

2a.  Select Void Payment.

OR

1b. Select the payment you wish to void.

2b.  On the Payment details screen, select Void Payment (in blue).

Note: If all or part of the payment has been allocated, Void Payment will not be an option.

3.  From Void Payment, select a Void Reason, then click VOID (in blue).

Note: Void Reasons must first be created in Billing Settings - Billing Codes. Click here to learn more about Billing Settings - Billing Codes.

4.  The Payment status for that particular item will now say Voided.  Both the Payments and Balance cards will update accordingly. 

 5.  The Payment details page will also update to reflect the Voided Status.

Refund Payment

Voiding a payment from the Payments card can be done in either of two ways.

1a. Select the checkbox of the payment you wish to refund, then click the action menu (ellipsis). From the action menu (ellipsis), select REFUND.

OR

1b. Click anywhere on the charge line for the payment you wish to Refund.

2.  The payment details screen will open.  Select Refund Payment (in blue).

3.  From the Refund Payment dialog, select a Refund Reason and Method, followed by REFUND (in blue).

Note: Refund Reasons must first be created in Billing Settings - Billing Codes. Click here to learn more about Billing Settings - Billing Codes.

4. The Payment status for that particular item will now say Refunded.  Both the Payments and Balance cards will update accordingly.  

The Payment details screen will update to reflect the Refunded Status.

Allocate Payment

The ability to allocate a payment can be found in multiple areas. Regardless of area, the steps required to allocate payment remain the same. All unallocated payments can be allocated. 

1a.  Select the payment via the patient checkbox, followed by the action menu (ellipsis).

2a.  From the action menu (ellipsis), select Allocate Payment.

OR

1b. Select the Payment you wish to Allocate. 

2b.  From the view Payment details page, select Allocate Payment (in blue).

3.  From the Allocation slideout, the open Charges will display at the bottom, selected by default.

Note: Open Charges who match the payment criteria will only be available to allocate.  Charge criteria include responsibility type (patient or payer) and an open balance to allocate to.  In this example, will be allocated a patient payment to a patient responsible charge.

4. Scrolling to the right on the bottom charge table, find the responsibility section that corresponds to the payment.  Update the Allocated column with the desired amount you wish to allocate.

Note: Charges can be fully, or partially allocated.

5. If you need to add an adjustment, decrease the Pt. Owed amount. Then select the adjustment code, followed by the adjustment amount.

6. Select Allocate (in blue).

7.  The payment details will update to reflect the allocation.
