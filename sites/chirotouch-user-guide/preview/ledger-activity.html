Patient Ledger - Activity 

The Activity card in the Ledger tab of a Patient Record displays all patient-related ledger activity, including Encounter details, Purchases, and Transaction History. 

Encounter Tab

When viewing an individual patient, under the Ledger tab, you can use the Activity card to see all patient encounters and related status. Users can filter results to quickly access encounter details.

  • Last Days Seen - select the number of days previous to today 
  • Date - add a date range
  • Providers - select one or more providers
  • Charge Status - select whether all, some, or none of the charges have been posted 
  • Invoice # - add the invoice number if it’s known
  • Encounter # -add the encounter number if it’s known 
  • Balance - add a minimum and/or maximum outstanding balance amount

View Encounter Details & Summary

1. The Activity card in the Ledger tab of an individual patient, will display pertinent details related to the encounter which includes:

  • Date - date of service of the encounter
  • Type - displays the type of encounter
  • Encounter - displays the encounter number and status, clickable hyperlink, opening encounter details 
  • Charge Status - displays the status of the charges associated with that encounter
  • Charges - displays charges total amount associated with that encounter
  • Balance - displays the balance and responsible parties of the charges associated with that encounter 
  • Outstanding - displays the outstanding balance of charges associated with that encounter

2.  Click the Encounter number (in blue hyperlink) to view the encounter details.

3.  The Encounter Summary slideout will display.

Collect Payment

1.  Select the encounter action menu (ellipsis). 

Note: Action menu options displayed will vary based on encounter and charge status. 

2. Select Collect Payment.

2.  From the Add Payment slideout, fill out the payment details, followed by POST (in blue).

3.  From the Posted Payment slideout, users have the ability to:

  • Allocate Payment - allocate this payment to open related charges
  • Print Receipt - opens up printable receipt in a new tab

Create Invoice

Invoices can also be created from the Charge section of the Patient Record. Click here to learn more about Charges.

1.  Select the encounter action menu (ellipsis) whose charges you wish to Create Invoice. 

2.  The Encounter will now reflect the newly-created invoice. Click on the blue invoice hyperlink to view the corresponding charges. 

View Charges From Encounters

Charges can also be viewed and edited from within the Charge section of the Patient Record. Click here to learn more about Charges.

1.  Select the encounter action menu (ellipsis) for the encounter you wish to View Charges.

2.  You will be routed to the view Charges page. You can edit, update, and make changes for the selected charges. Click here to learn more Charges.

Remove Posted Charge

Once a charge has been posted, you will need to remove the charge from the invoice in order to unpost the charge. Before removing a charge from an invoice, verify the charge is not associated with a claim or draft.

Note: If the Encounter has been signed, you will need to reopen the encounter to remove the charge.

1. Click the action menu (ellipsis) and View Charges for the invoice. This encounter is open and you will be able to unpost this charge.  

2.  Check the charge you'd like to Remove From Invoice.  Notice the Unpost is grayed out.  You must first remove the charge from the invoice before you can unpost.

3.  You will receive a prompt to acknowledge you want to remove the charge.  Select Yes.

4.  Close out of Charges (x in the top right corner of screen).  Now you will see the charge is no longer invoiced. Use the action menu (ellipsis) to open the charge again.

5. Now check the box for the charge you want to unpost.  The Unpost option is now available on the Charges page.

Note: Unpost is not available if the chart note is signed.  You will need to return to the Encounter and reopen the note.

6.  When you Unpost the charge, you will receive a prompt acknowledging the charge will be removed. Click Yes.

7.  You can see that the charge is no longer posted.  The charge will still appear in the encounter and may be completely removed from there.

Purchases Tab

The Purchases section within the Activity card displays all patient purchases. You can review patient purchases, collect payments, view receipts and review invoices.

Use the filters on the Purchase tab to quickly find what you need.

  • Purchase Type - type in a procedure or charge
  • Date - add a date range
  • Invoice # - add the invoice number if it’s known
  • Not Invoiced - check the box if the charge has not been invoiced yet
  • Encounter # - add the encounter number if it’s known 
  • Balance - add a minimum and/or maximum outstanding balance amount

View Charges

1.  Select the purchase action menu (ellipsis), followed by View Charges.

2.  You will be routed to the view Charge details page, where users can edit, update, and make changes for the selected charges. Click here to learn more Charges.

Collect Payment

1.  Select the purchase action menu (ellipsis), followed by Collect Payment.

2.  From the Add Payment slideout, fill out the payment details, followed by POST (in blue).

3. The charge will now be updated to reflect the payment.

View Receipt

Invoices can also be viewed from within the Charge section of the Patient Record. Click here to learn more about Charges.

1.  Select the encounter action menu (ellipsis) whose receipt you wish to view, followed by View Receipt. 

2. A printable receipt will open in a new browser tab.

Transaction History

The Transaction History section in the Activity card displays all patient transaction history.

Use the filters on the Transaction tab to sort through and find specific charges.

  • Amount - transaction amount
  • Date - time frame for the charges you wish to view
  • Encounter # - add the encounter number if it’s known
  • Invoice # - add the invoice number if it’s known
  • Not Invoiced - check the box if an invoice has not yet been generated
