Practice Ledger - Payments 

The Payments card in the Practice Ledger displays all patient and payer payments for the entirety of your practice. Here, users have the ability to manage all payments which includes adding, editing, voiding, refunding, and allocating. 

Use the filters to narrow your search.

Add Payment

In the Practice Ledger, you have the ability to add payer payments that are not specific to one patient. You can then allocate these payments across multiple patients.

1.  Select Add New Payment (in blue).

2.  In the Add Payer Payment slideout, fill in all relevant details.

  • Payment Type -select a payment code
  • Payer - select the payer
  • Amount - enter the payment amount
  • Transaction Date - select the payment date
  • Dates of Service - enter them if known
  • Payment Method - choose from:
    • Check
    • Direct Deposit 
    • Debit or Credit Card
    • ERA
  • Reference ID - refers to the payer
  • Authorization/Check/EFT # - refers to patient payment methods (ex: check number, credit card authorization)
  • Note - add details if necessary

4.  Select POST (in blue).

5.  From the Posted Payment slideout, you have the ability to:

  • Allocate - allocate this payment to an open charge(s)
  • Void - void the payment
  • Close - close the window and return to Payments

Payment Details

The Payment page displays all payment-related information. From this page, you can take several actions related to payments.

1. Select the payment you wish to view.

2.  From the Payment details, view the available actions indicated (in blue):

  • Void - voids the individual payment 
  • Refund Payment - refunds the individual payment
  • Allocate Payment - allocates the payment to related open charges
  • Receipt - only available for patient payments 
  • Edit icon - allows you to edit specific payment details

Note: Actions in blue indicate that they are available to be taken against the payment. Any actions that are grayed out are not available. The actions available for a particular payment are dependent on the type of payment (patient or payer) and the payment status (unallocated, allocated, voided, refunded).

Edit Payment

1. Select the payment you wish to edit.

2.  From the payment details, select the edit icon (in blue).

3.  In the Update Patient Payment slideout, edit the necessary fields. Click SAVE (in blue).

Note: Editable fields are dependent on the payment method (cash, check, direct deposit, debit or credit card, or ERA) and whether or not all or part of the payment has been allocated.

  • Payment Type – always editable
  • Payer – editable for payer payments only; only editable if none of the payment has been allocated
  • Amount – editable if none of the payment has been allocated
  • Payment Method – editable unless the payment was made with a debit or credit card and if none of the payment has been allocated
  • Transaction Date – editable unless the payment was made with a debit or credit card
  • DOS From – always editable; only available if it is a payer payment
  • DOS To – always editable; only available if it is a payer payment
  • Reference ID – always editable; only available if it is a payer payment
  • AUTH/CHECK/EFT – editable if the payment was made by check, direct deposit, or ERA
  • Card Sale ID – never editable; only viewable if the payment was made with a debit or credit card

Note: In order to update the payment method to a card transaction, select Debit or Credit Card in Payment Method, then proceed through the steps of making the payment.

Void Payment

Voiding a payment from the Payments card can be done in either of two ways.

1a. Select the payment checkbox you wish to void, followed by the action menu (ellipsis). Then select Void Payment.

OR

1b. Select the payment you wish to Void. 

2.  From the Payment details page, select Void Payment (in blue).

Note: If all or part of the payment has been allocated, Void Payment will not be an option.

3.  From the Void Payment dialog, select a Void Reason followed by VOID (in blue).

Note: Void Reasons must first be created in Billing Settings - Billing Codes. Click here to learn more about Billing Settings - Billing Codes.

4.  The Payment status will update to Voided, updating both the Payments and Balance card accordingly. 

The Payment details screen will update to reflect the voided status.

Refund Payment

Refunding a Payment from the Payments card can be done in two ways.

1a. Select the payment checkbox, followed by the action menu (ellipsis). Then select Refund Payment.

OR

1b. From the payment details, select the payment you wish to refund.

2. On the Payment details screen, select Refund Payment (in blue). 

3. In the Refund Payment dialog, select a Refund Reason and a Refund Method, then click REFUND (in blue).

Note: Refund Reasons must first be created in Billing Settings - Billing Codes. Click here to learn more about Billing Settings - Billing Codes.

4. The payment status will update to Refunded.  Both the Payments and Balance cards will update accordingly. The Payment details page will update to reflect the refunded status.
